# i18n

Stack: **i18next v25 + react-i18next**. Romanian is the default/fallback language.

## Rules

- All user-visible text: `const { t } = useTranslation()` then `t('domain.key')`
- Both `t` and `translate` are valid aliases — match the convention in the file you're editing
- Never hardcode strings in JSX — hookify blocks it
- New keys: add to **both** `src/locales/metadata/en.json` and `src/locales/metadata/ro.json`
- Dynamic values: i18next interpolation `t('key', { count: 5 })` with `{{count}}` in JSON

## Default Language

`fallbackLng: 'ro'` — Romanian is the default. User language preference is persisted in AsyncStorage. Do not assume English is the fallback.

## Pattern

```tsx
const { t } = useTranslation();

// Static key
<Text>{t('login.title')}</Text>

// With interpolation
<Text>{t('login.passwordMinLength', { count: 8 })}</Text>
```

## JSON Key Structure

Keys use dot notation, organized by feature domain:

```json
{
  "login": {
    "title": "Autentificare",
    "phoneNumber": "Număr de telefon",
    "passwordMinLength": "Parola trebuie să aibă minim {{count}} caractere"
  },
  "dashboard": {
    "welcome": "Bun venit"
  }
}
```

Add keys under the matching domain section. Keep English and Romanian in sync.
